## Directory Purposes

**`backend/`:**
- Purpose: Flask API server, LSTM+Transformer model, explainability engine
- Contains: Python modules, dependencies, runtime data/models
- Key files: `app.py` (8 endpoints), `model.py` (393 lines, architecture), `explainer.py` (343 lines, explanations)
- Executed by: `python app.py` (starts server on port 5000)

**`backend/data/`:**
- Purpose: Runtime dataset storage (created automatically)
- Contains: Single active CSV (`traffic.csv`) during session
- Generated by: `/upload` endpoint (user-provided) or `/generate-sample` endpoint (synthetic)
- Cleared by: Manual deletion; replaced on new upload

**`backend/saved_models/`:**
- Purpose: Persist trained model weights and scaler state across sessions
- Contains: `traffic_model.pth` (PyTorch weights) and `traffic_model_scaler.pkl` (MinMaxScaler)
- Generated by: `/train` endpoint after successful training
- Loaded on: Flask startup (if exists) to avoid retraining

**`frontend/`:**
- Purpose: Web-based user interface for the prediction pipeline
- Contains: Single-page HTML, vanilla JS, CSS3
- Entry point: `index.html` (open in browser)
- No build step; served directly or via simple HTTP server

**`data/`:**
- Purpose: Source/default datasets for quick testing
- Contains: Two CSV files (large production dataset + small 168-hour sample)
- Copied to: `backend/data/traffic.csv` on Flask startup if default exists

**`.planning/codebase/`:**
- Purpose: GSD codebase analysis documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, STACK.md, INTEGRATIONS.md, CONCERNS.md
- Not part of runtime; consumed by planning tools

- `frontend/index.html`: HTML entry point - open in browser to start UI

  - `get_attention_weights()`: Extract temporal attention from transformer (lines 286-336)
  - `_create_sequences()`: Sliding window creation (lines 338-346)
  - `save/load()`: Persistence (lines 348-393)

- `backend/explainer.py`: Explainability functions (lines 1-343)
  - `explain_prediction()`: Main orchestrator (lines 9-66)
  - `calculate_feature_importance()`: Perturbation-based (lines 138-165)
  - `generate_counterfactuals()`: What-if scenarios (lines 168-242)
  - `generate_text_explanation()`: Natural language (lines 245-309)
  - `_get_temporal_attention_safe()`: Attention extraction with fallback (lines 69-105)

- `backend/app.py`: Flask routes (lines 43-292)
  - `@app.route('/')`: Home/documentation (line 43-53)
  - `@app.route('/upload', methods=['POST'])`: File upload with Kaggle format auto-detection (lines 77-121)
  - `@app.route('/train', methods=['POST'])`: Model training (lines 124-153)
  - `@app.route('/predict', methods=['POST'])`: Prediction + explanation (lines 156-198)
  - `@app.route('/status', methods=['GET'])`: Status check (lines 201-220)
  - `@app.route('/data-stats', methods=['GET'])`: Dataset statistics (lines 223-238)
  - `@app.route('/generate-sample', methods=['POST'])`: Synthetic data generation (lines 241-291)
  - `convert_kaggle_format()`: Greenshields traffic model (lines 56-74)

- `frontend/script.js`: Frontend logic (~400+ lines)
  - Event initialization (DOMContentLoaded)
  - File upload handling with drag-drop
  - API calls to all 6+ endpoints
  - Chart.js visualization (feature importance, temporal attention)
  - Status indicator updates

**Testing:**

## Naming Conventions

**Files:**

- **Python modules:** lowercase with underscores (`model.py`, `explainer.py`, `app.py`, `prepare_kaggle_data.py`)
- **HTML:** single file `index.html` (convention)
- **CSS:** single file `style.css`
- **JS:** single file `script.js`
- **Data files:** lowercase with underscores (`traffic_final_clean.csv`, `sample_traffic.csv`)
- **Model files:** `traffic_model.pth` + `traffic_model_scaler.pkl` (matching pair)

**Directories:**

- **Functional:** `backend`, `frontend`, `data` (purpose-based)
- **Output:** `saved_models`, `__pycache__` (underscore prefix for generated)

**Python Symbols:**

- **Classes:** PascalCase (`TrafficLSTM`, `PositionalEncoding`, `TransformerEncoderLayerWithWeights`)
- **Functions:** snake_case (`explain_prediction`, `calculate_feature_importance`, `generate_counterfactuals`, `convert_kaggle_format`)
- **Constants:** UPPER_SNAKE_CASE (`FREE_FLOW_SPEED`, `ROAD_CAPACITY`, `DATA_DIR`, `API_URL`)
- **Variables:** snake_case (`trained_model`, `input_data`, `epoch_loss`, `dataset`)

**JavaScript Symbols:**

- **Functions:** camelCase (`checkStatus`, `uploadFile`, `generateSampleData`, `setupFileUpload`, `fetchDataStats`)
- **DOM IDs:** kebab-case (`uploadArea`, `fileUpload`, `uploadStatus`, `dataStatus`, `modelStatus`)
- **Variables:** camelCase (`featureChart`, `attentionChart`, `API_URL`)

**HTML IDs and Classes:**

- **Semantic:** `uploadSection`, `trainSection`, `predictSection` (matches step flow)
- **Utility classes:** `btn`, `btn--primary`, `btn--ghost`, `card`, `card--stagger-1`, `pill`
- **Status indicators:** `dataPill`, `modelPill`, `dataStatus`, `modelStatus`

## Where to Add New Code

**New Feature (e.g., add hourly data aggregation):**

- **Primary code:** `backend/model.py` - add preprocessing method to TrafficLSTM
- **Tests:** Create `backend/tests/test_model.py` (currently no test directory)
- **API route:** Add endpoint in `backend/app.py` if user-facing
- **Frontend:** Add form/button in `frontend/index.html` + handler in `frontend/script.js`

**New Component/Module (e.g., new attention visualization):**

- **Implementation:** Create `backend/attention_viz.py` if backend logic, or JS module in `frontend/`
- **Integration:** Import into `backend/app.py` or `backend/explainer.py`; call from route or explanation pipeline
- **Persistence:** If requires model changes, update TrafficLSTM in `backend/model.py`

**Utilities (e.g., shared helpers for data validation):**

- **Shared helpers:** Add to existing modules (`backend/explainer.py` has utility functions like `_describe_attention_pattern()`)
- **Data validation:** Add to `backend/app.py` (routes already contain validation logic)
- **Frontend utilities:** Add to top of `frontend/script.js` (single-file approach)

**Tests:**

- Create directory: `backend/tests/` (does not exist)
- Test file naming: `test_*.py` or `*_test.py` (pytest convention)
- Test data: Use `data/sample_traffic.csv` for fixtures
- Run: `pytest backend/tests/` (after setup)

## Special Directories

**`backend/saved_models/`:**
- Purpose: Persist trained model state
- Generated: Yes (created by `/train` endpoint)
- Committed: No (in .gitignore via `__pycache__` pattern, but explicit model exclusion may be needed)
- Cleanup: Can be safely deleted; model will retrain on next `/train` call

**`backend/data/`:**
- Purpose: Working dataset during session
- Generated: Yes (created by `/upload` or `/generate-sample`)
- Committed: No (runtime data)
- Cleanup: Can be safely deleted; will be regenerated on next upload or Flask startup copies default

**`backend/__pycache__/`:**
- Purpose: Python bytecode cache
- Generated: Yes (automatically by Python)
- Committed: No (.gitignore)
- Cleanup: Safe to delete; will regenerate

**`.git/`:**
- Purpose: Version control repository
- Generated: Yes (via `git init`)
- Committed: Always (internal git structure)

**`.planning/`:**
- Purpose: GSD planning artifacts (analysis documents)
- Generated: Yes (by GSD analysis tools)
- Committed: Yes (tracked for collaboration)
- Location: Separate from source code

## Architecture Pattern Summary

- **Layered:** 3 tiers (UI → API → ML)
- **Monolithic:** Single Flask server, single model instance
- **Synchronous:** Request-response (no async/await, no queues)
- **Stateful:** Models and data persist on disk; loaded on startup
- **No framework bloat:** Vanilla JS, minimal Flask, PyTorch only for ML
